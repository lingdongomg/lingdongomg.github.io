高并发秒杀系统

介绍

学习完SpringBoot,实际上只是大致过了一遍，很多注释和深入的原理都没怎么看懂，跟着书上的内容试着做一下这个秒杀项目，书名为《一步一步学SpringBoot》从这个项目中巩固所学习的内容。秒杀架构的设计主要思想有以下四点：

  1. 缓存
    将部分业务逻辑迁移到内存的缓存或者Redis中，从而极大的提高并发读效率。
  2. 削峰
    秒杀开始的一瞬间会有一个瞬间流量峰值，如果使这个瞬间的流量峰值变得平缓是成功设计秒杀系统的关键。要实现流量的削峰填谷，一般的方法是采用缓存和MQ中间件。
  3. 异步
    将同步业务设计成异步处理的任务，以提高网站的整体可用性。
  4. 限流
    秒杀库存量一般很少，只有少部分用户才能秒杀成功，所以需要限制大部分用户流量，只准少量用户流量进入后端服务器。

搭建开发环境

创建SpringBoot项目时选择创建Spring Initializr（初始化）项目，IDEA会自动配置默认的SpingBoot服务器URL：start.spring.io

pom.xml

SpringBoot项目和Maven项目很像，有一个pom.xml文件，在其中已经自动添加了几个依赖和插件，除此之外该项目中还额外手动添加了以下依赖。

JPA（Java Persistence API）是Sum公司官方提出的Java持久化规范，不提供实现，JPA的主要实现有Hibernate、EclipseLink、OpenJPA等。
Spring Data JPA的顶层接口是Repository，是一个空类，它的子类有CrudRepository、JpaRepository、PagingAndSortringRepository等。我们在使用时都是通过实现JpaRepository（包括增删改查分页接口）加上自己的业务方法。

thymeleaf是模板引擎，拥有丰富的标签语言和函数，在SpringBoot页面开发设计中一般采用thymeleaf编写html文件进行页面开发。

redis缓存，是被企业广泛使用的缓存。
使用方法：先在pom在加入redis依赖，然后在application.properties文件中添加配置信息（包括redis数据库(0)、服务器地址（localhost）、端口号（6379）、密码（空））。
redis的使用：在启动项目时会在监听器类中插入代码，将数据存放到redis缓存中，之后在service类中注入RedisTemplata对象就可以在缓存中进行相关操作了。redis的常用操作有：

  • redisTemplate.opsForList().leftPushAll(ALL_USER,x) 查询缓存中所有用户数据,若ALL_USER键不存在，会创建该键及与其关联的List，之后再将参数中是x从左到右依次插入。
  • redisTemplate.opsForList().range(ALL_USER,0,-1) 取链表中的全部元素，0代表第一个元素，-1代表最后一个元素。

JMS（Java Message Service）Java消息服务，是一组Java应用程序接口，提供消息的创建、发送、读取等一系列服务。JMS支持两种消息发送和接受模型，一种为P2P，基于队列的消息生产者发送消息到队列，消息消费者从队列中接受消息，队列的存在时消息的异步传输成为可能。另一种为Pub/Sub,（Publish/Subscribe）发布-订阅模型。定义了如何向一个内容节点（Topice）发布和订阅消息。该模型在消息的一对多广播时采用。

AyUser.java

@Entity、@Table、@Id、@GeneratedValue 都是JPA的注解

  • @Entity说明这个class是实体类，并且使用默认的orm规则，即class名即数据库表中表名，class字段名即表中的字段名
  • 如果想改变这种默认的orm规则，就要使用@Table来改变class名与数据库中表名的映射规则，@Column来改变class中字段名与db中表的字段名的映射规则
  • @Id 标注用于声明一个实体类的属性映射为数据库的主键列。该属性通常置于属性声明语句之前，可与声明语句同行，也可写在单独行上。
  • @GeneratedValue 用于标注主键的生成策略，通过strategy 属性指定。默认情况下，JPA 自动选择一个最适合底层数据库的主键生成策略：SqlServer对应identity，MySQL 对应 auto increment。

ActiveMQ

activemq中间件，MQ全称为MessageQueue消息队列，是一个消息的接受和转发的容器，可用于消息推送，ActiveMQ是Apache提供的一个开源的消息系统，完全由Java实现，因此能很好的支持J2EE提出的JMS规范。
使用方法：先引入依赖，再到application.properties配置文件中添加ActicveMQ配置，包括

  • 端口号 61616，
  • memory true,
  • enabled false,
  • 信任所有的包trust-all true。

之后需要创建Producer类和Consumer类，在Producer类中注入JmsMessagingTemplate类，该类是发消息的工具类，并创建方法sendMessage,方法中调用注入类的convertAndSend方法。
在Consumer类中使用@JmsListener标签来监听一个队列，并创建receiveQueue方法处理接收到的信息。
在实际使用时注入创建的Producer类，然后创建destination对象（创建消息队列），再往队列中发送消息即可。
