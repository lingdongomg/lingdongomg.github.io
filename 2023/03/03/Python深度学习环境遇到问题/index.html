Python深度学习环境遇到问题

近期在研究毕设，毕设内容为深度学习图像增强相关，先找相关内容，在GitHub上找到 (https://github.com/sczhou/CodeFormer) 这个项目发现很不错，便想将代码拷贝下来到本地跑一下。

anaconda环境

第一个问题为python环境的配置，之前编写python代码都是在windows系统上，使用pycharm工具完成，对于python环境的管理可以直接在pycharm上设置，也可以直接采用虚拟环境，但这样配置环境其实一直有问题，就是在需要安装依赖的时候命令行的默认pip并不是自己这个虚拟环境的，容易出问题，得在pycharm安装包，而pycharm不知道什么原因安装包检查会失败。之前用着就很难受，现在在Ubuntu系统上运行python项目，越来越难受了，所以放弃用pycharm来管理环境。

所以最好的处理方法还是利用anaconda来进行环境的管理，这个安装好后的conda指令也很简单方便。

  1. conda –version #查看conda版本，验证是否安装

  2. conda update conda #更新至最新版本，也会更新其它相关包

  3. conda update –all #更新所有包

  4. conda update package_name #更新指定的包

  5. conda create -n env_name package_name #创建名为env_name的新环境，并在该环境下安装名为package_name 的包，可以指定新环境的版本号，例如：conda create -n python2 python=python2.7 numpy pandas，创建了python2环境，python版本为2.7，同时还安装了numpy pandas包

  6. conda activate env_name #切换至env_name环境

  7. conda deactivate #退出环境

  8. conda info -e #显示所有已经创建的环境

  9. conda create –name new_env_name –clone old_env_name #复制old_env_name为new_env_name

  10. conda remove –name env_name –all #删除环境

  11. conda list #查看所有已经安装的包

  12. conda install package_name #在当前环境中安装包

  13. conda install –name env_name package_name #在指定环境中安装包

  14. conda remove – name env_name package #删除指定环境中的包

  15. conda remove package #删除当前环境中的包

  16. conda create -n tensorflow_env tensorflow

conda activate tensorflow_env #conda 安装tensorflow的CPU版本

  1. conda create -n tensorflow_gpuenv tensorflow-gpu

conda activate tensorflow_gpuenv #conda安装tensorflow的GPU版本

  1. conda env remove -n env_name #采用第10条的方法删除环境失败时，可采用这种方法

连接超时

异常描述：

WARNING: Retrying (Retry(total=3, connect=None, read=None, redirect=None, status=None)) after connection broken by ‘NewConnectionError(‘<pip._vendor.urllib3.connection.HTTPSConnection object at 0x7fe4d1313850>: Failed to establish a new connection: [Errno 101] Network is unreachable’)’: /simple/cython/

ERROR: Could not find a version that satisfies the requirement cython (from versions: none)

ERROR: No matching distribution found for cython

先是跳出四条WARNING，内容是HTTPS连接不可达，后面两条ERROR，查了好久资料，还是用Google查的，又是修改防火墙又是改时间又是信任pip的。最后在csdn找到解决方案。

问题出现主要还是网络问题，用pip默认的地址不可达其实换个地址就可以了（挂代理似乎没用），比如换成国内的清华源或者豆瓣镜像。

解决方法为：创建文件：~/.pip/pip.conf

在该文件内加上下面内容就解决问题了

[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
[install]
trusted-host=mirrors.aliyun.com

同理 conda在执行install的时候也会报http连接错误，相同的方法修改配置文件，将源改为清华镜像即可
