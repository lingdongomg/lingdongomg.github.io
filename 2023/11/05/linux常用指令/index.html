4. journalctl 查看系统日志

系统如果出现问题，那么就需要查看日志。linux系统下所有的日志都存放在/var/log目录下。查看日志比较简单的方法是使用journalctl，他会把所有的日志都聚合到终端下，方便查看。

journalctl也可以通过管道和其他的程序配合来使用。

搜索文件

grep rk3368 -rn device/rockchip/xxx_name

在当前目录下的device/rockchip/xxx_name文件夹中递归地搜索包含”rk3368”关键字的文件，并显示匹配行的行号和内容。它用于快速查找与指定关键字相关的内容或文件。

grep -rn NAVICATION_SHOW

查看当前目录的所有子目录文件中名字带有NAVICATION_SHOW的文件

Linux 使用find指令 查找文件夹下所有子文件夹里面是否存在一个文件

find . -type f -name "a.txt"

.: 表示从当前目录开始进行查找。

-type f: 限定搜索结果为文件类型。

-name “a.txt”: 指定要查找的文件名为a.txt。通过使用-name选项和相应的模式匹配来查找文件。

批量重命名

rename 's/rk3288/new_name/' /path/to/directory/*
批量将指定文件夹下所有文件 名字带有rk3288的改为新名字 new_name

sed -i ‘s/rk3288/new_name/g’ /path/to/directory/*
批量将一个文件夹中所有文件内容中带有rk3288的改为我想要的新名字 new_name
-i：这是sed命令的选项之一。它表示直接在原始文件中进行修改。当使用-i选项时，须提供一个后缀作为备份文件的扩展名。例如，如果你想要保留备份文件，可以使用-i.bak。
s/old/new/g 代表替换操作。其中：
s/：表明这是一个替换操作。
rk3288：是需要被替换的模式或字符串。
new_name：是替换后的新字符串。
g：表示全局替换，即一行中所有匹配到的模式都会被替换。

批量将一个文件夹中所有文件和子文件夹中所有文件内容中带有rk3288的改为我想要的新名字 new_name

find /path/to/folder -type f -exec sed -i 's/rk3288/new_name/g' {} +

批量将一个文件夹中所有文件和子文件夹中所有文件内容中带有rk3288的改为我想要的新名字 new_name

find /path/to/folder：使用find命令来搜索/path/to/folder路径下的所有文件和子文件夹。

-type f：-type选项用于指定要搜索的文件类型。在这种情况下，f表示只搜索普通文件，而不包括目录或符号链接。

-exec sed -i ‘s/rk3288/new_name/g’ {} +：对于每个找到的文件，使用sed命令进行替换操作。

sed命令是一个流编辑器，可以用来对文本进行各种操作，其中-i选项用于直接修改文件，并且’s/rk3288/new_name/g’是替换操作的表达式。

s/rk3288/new_name/g表示将文件中所有出现的 “rk3288” 替换为 “new_name”。s是替换操作的命令，g表示全局替换，即替换所有匹配项而不只是第一个。

{}表示当前找到的文件，+表示将多个文件一次传递给sed命令，以便高效处理。
