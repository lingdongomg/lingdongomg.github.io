Android.mk怎么写.md

0、小小的示例

1、指定文件

LOCAL_SRC_FILES 用于指定要编译的源代码文件。$(call all-java-files-under, src) 是一个 GNU Make 函数调用，用于搜索指定目录下的所有 Java 源代码文件。

LOCAL_AIDL_INCLUDES 变量用于告诉构建系统在哪里寻找 AIDL 文件，以便在构建过程中将这些 AIDL 文件编译成 Java 文件。

一般来说aidl文件放在LOCAL_AIDL_INCLUDES 中即可，无需额外在LOCAL_SRC_FILES 中添加，不过也有时候直接在LOCAL_SRC_FILES 添加后，也可以不用在LOCAL_AIDL_INCLUDES 中添加，不过两个都加进去也无妨。

2、想要编译成什么结果

Android.mk文件可以将模块编译为：apk、jar包、二进制文件 等等，想要不同的文件，则最后一个include则变为不同的类型即可。

include $(BUILD_STATIC_JAVA_LIBRARY)：这个指令告诉构建系统根据指定的 LOCAL_MODULE 和 LOCAL_SRC_FILES 编译 Java 源代码文件，并生成静态 Java 库文件。静态 Java 库文件是一个包含 Java 类文件的 JAR 文件，可以用于在其他应用程序中引用和使用。该指令一般需要和上面这条指令一起使用，用于屏蔽Jack编译器。

include $(BUILD_JAVA_LIBRARY)：这个指令类似于 BUILD_STATIC_JAVA_LIBRARY，但生成的是一个用于 Android 应用程序的 JAR 文件，而不仅仅是库文件。这个 JAR 文件可能包含应用程序的代码、资源和其他文件，可以用于构建 APK。

include $(BUILD_EXECUTABLE) 告诉构建系统将模块编译为可执行文件。

将模块编译为apk文件

3、权限控制

设置该属性后，模块中的 API 会被视为私有平台 API。这意味着只有系统级别的应用程序或者使用系统签名的应用程序才能访问您的模块中的这些 API，普通的第三方应用程序将无法访问它们。

LOCAL_CERTIFICATE 是一个变量，用于指定签名密钥的别名，以及要用于签署模块的证书。其中，platform 是一个预定义的别名，代表了 Android 平台密钥，设为它后模块将使用系统默认的签名密钥进行签名。

LOCAL_PRIVILEGED_MODULE := true： 这个选项用于标识一个模块是否是特权模块。特权模块是那些需要更高权限的模块，例如系统服务、系统框架等。设置为 true 表示该模块被认为是特权模块，拥有更高的权限级别。

5、APK模块才用到的

LOCAL_PACKAGE_NAME 指定了 APK 的包名。

LOCAL_MANIFEST_FILE 指定了 AndroidManifest.xml 的路径。

LOCAL_RESOURCE_DIR 指定了资源文件的路径
