Docker-compose前后端项目部署笔记.md

简介

项目：补丁仓库，用于存放上层和嵌入式人员的补丁，提供查询功能，便于复用。

框架：基于 GitHub 开源项目 RuoYi

  • 前端：Vue3 （Element-Plus）
  • 后端：SpringBoot + SpringSecurity
  • 数据库：Mysql8.0
  • 中间件：Redis
  • 部署方式：Docker-compose

部署过程

首先划分模块，根据技术选型发现所需要的容器服务有：

1、MySQL

2、Redis

3、Nginx

4、Openjdk-8

根据四个模块编写 docker-compose.yml 进行

构建指令：docker-compose build

在构建的时候如果本地有镜像默认会使用本地的镜像，但本地的镜像会被文件中的CMD修改，所以为了使用未被修改的镜像，需要使用：docker-compose build --pull 该指令会从远程仓库拉取镜像

启动指令：docker-compose up -d

关闭指令：docker-compose down

docker-compose.yml

其中的services就是 docker-compose 中的服务，其中包含了四个服务。以及四个服务的配置

在其中可以像启动docker一样指定路径映射、端口映射、采用镜像等

比较重要的是一个 docker-compose 中， 所有的服务都默认在同一个网段下，但不是同一个ip

四个服务的 ip 分别为 192.168.240.2 、192.168.240.3 、 192.168.240.4 、192.168.240.5

查看容器的 ip 地址的指令为 ：docker inspect <container_id> | grep IPAddress

所以容器之间不能直接用 localhost 进行调用，可以通过服务名进行调用，例如后端在连接数据库时，所填写的 url 变成了 url: jdbc:mysql://hukaichao-mysql:3306/techvision 所以从这里的网络就会衍生出一系列问题。

容器进行端口号映射之后，宿主机可以用 localhost+端口号 来调用容器内的服务，但是容器和容器之间是不能的。

mysql-dockerfile

在拉取 mysql 的镜像之后，将sql语句放到容器中的 /docker-entrypoint-initdb.d/ 目录下，该目录下的sql语句会在容器初次启动时自动执行。mysql的文件映射如下：

./mysql/my.cnf:/etc/my.cnf

./mysql/logs:/var/log/

./mysql/data:/var/lib/mysql

第一项 my.cnf 为 MySQL的配置文件，其中指定了 mysql异常的日志路径：log-error = /var/lib/mysql/error.log

第二项为 MySQL的日志信息

第三项为 MySQL的数据文件，其中保存的为 binlog 文件 即二进制日志文件，记录了MySQL的所有执行语句，可以根据该文件还原数据库。

redis-dockerfile

redis 和 MySQL类似，挂载文件同样为配置文件和数据文件，这些是数据库中需要备份的

redis.conf 文件主要修改的地方有：

  • 注释掉绑定的 ip # bind 127.0.0.1
  • 关闭保护模式 protected-mode no
  • 为redis设置密码 requirepass Techvision8888

这样才能允许除了 localhost 之外的 ip 访问 redis 数据库

​ root 指向生成的静态资源文件。

​ location /prod-api/ 是对所有以 /prod-api/开头的请求生效，将其进行一次代理，代理后再将请求发送到 http://hukaichao-server:8888/ 这个服务器端。目的是为了避免前端采用 post 发送请求，Apache、IIS、Nginx 等绝大多数web服务器，都不允许静态文件响应POST请求，否则会返回“HTTP/1.1 405 Method not allowed”错误

​ 有的解决方式是直接用 error_page 405 =200 $uri; 这一行相当于是将post请求转化为了get请求，但是我这个项目依然不行，所以只能采用代理的方式解决。

部署时遇到的问题

装docker-compose

在服务器上，装好docker后是没有docker-compose的，这个需要单独安装，如果使用apt install安装docker-compose 安装的版本是非常低的 以至于无法启动docker-compose。所以这时可以去GitHub上下载新版本的docker-compose二进制文件：https://github.com/docker/compose/releases?expanded=true
下载完成后复制到下面路径并重命名文件：
sudo cp docker-compose-linux-x86_64 /usr/local/bin/docker-compose

将可执行权限应用于二进制文件：
sudo chmod +x /usr/local/bin/docker-compose

创建软链：
sudo ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose

测试是否安装成功：
docker-compose version

报错

这个问题出在Linux缺少一个密码管理包gnupg，它用于加密，我们在登录时需要这个包将密码加密后才能完成，因此直接安装
sudo apt install gnupg2 pass
