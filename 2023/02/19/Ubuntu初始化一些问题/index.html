Ubuntu初始化一些问题

之前学习Linux课程的时候上课采用的操作系统为Redhat

后面Hadoop相关课程采用的操作系统都是Centos

自己折腾玩的云服务器也都是Centos操作系统

这一次因为一个深度学习的环境是在Ubuntu上运行的，特意试了一下Ubuntu操作系统，刚刚使用的时候遇到不少坑，花了点时间才解决

采用的操作系统：Ubuntu 20.04

采用的虚拟机软件：VMware Workstation 16pro

令人满意的是这个系统自带了python 3.8.5，算是很好用的python版本，不需要额外安装了。

1、无法安装工具

Ubuntu安装工具采用的包管理工具是apt，安装工具的指令一般为：apt-get install [包名]

想要安装vim工具，因为Ubuntu自带的vi太难用了，和centos的vi有很大差距，用着很难受。但会抛出异常没法安装这个包。

导致这个的原因是apt的版本太旧，需要升级，升级的指令为：apt-get -y update

但运行这个指令依然会报错，查之后发现是因为源有问题，所以没法升级，需要修改文件 /etc/apt/sources.list

所以得用vi将国内一些镜像源的地址给复制到 sources.list这个文件中才能完成更新，接着才能安装包。

2、Xshell无法连接虚拟机

因为Xshell可以很方便的复制文本内容到Linux的文档中，所以习惯使用Xshell连接虚拟机。

连接虚拟机首先需要有虚拟机的ip地址，我最开始设置的为采用主机模式，可能有一部分这个原因。查资料说需要将模式修改为NAT模式，然后在Ubuntu中查ip地址。Ubuntu查ip地址的指令为：ip a

找到ip地址，可以互相ping通，Xshell依然连接不上，是因为Ubuntu默认没有开启SSH，这个需要安装包：openssh-server

安装完成后输入指令 /etc/init.d/ssh start 来开启SSH服务

开启后Xshell能连接密钥，但我连接root用户时输入密码一直显示连接被拒绝，发现如果要连接root用户的话还有一个地方得修改，即 /etc/ssh/sshd_config 在里面找到PermitRootLogin，将其后面的 prohibit-password 为 yes，最后才成功连接。

另外 还有一些小问题：
没有root用户，需要手动创建
sudo passwd root

自带的Python是没有pip的，需要用apt install python3-pip

不过最后还是没有成功

VMware workstation的Linux虚拟机无法使用主机自带的独立显卡，我创建该虚拟机就是为了运行深度学习的代码，所以不方便使用，转战WSL

现在安装WSL时默认的是WSL2，不需要修改版本

WSL2安装完成会自己配置一个Ubuntu系统，我的版本是22.04，挺新的，在WSL安装的这个Ubuntu里面没有我上面说的Ubuntu初始化的一堆问题，好用了不少，并且自带Python3.10
