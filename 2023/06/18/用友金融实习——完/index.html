用友金融实习——完

离职

其实有打算在用友金融干到转正的，毕竟已经干了两个半月了，只剩最后一个月就能转正了。不过现在已经到了九月份，开始秋招了，这一个月对我而言可能至关重要，还是决定先全力准备秋招，于是再九月一号正式离职。在职时间77天。

导师

我始终觉得我的导师是我工作上遇到的第一个贵人，尽管可能没有亲自手把手教导技能与知识，但是对我的培养方案和方向上的指点让我的收益太大了。
上班第一天告诉我实习生不要求加班，到点就可以走。在布置任务时不会给我过大的压力，也从来没有过压榨我的劳动力。当我把一个任务完成的时候，我的导师不会立刻给我分配下一个任务，而是将我多余出来的时间交由我来进行自我学习，这一点也让我在实习的时候看完了三本书，第四本看了一版，还看B站的视频学习Mybatis、MybatisPlus、SpringBoot、docker、juc等等东西，收益很大，感觉这才是真正的实习，在工作中学习。和我一同在用友金融实习的室友所在的小组就没这么幸运了，感觉他们的导师会给他们布置很大的任务，而那些工作又是重复的利用工具进行生成和增删改查等等操作，能学习到的东西甚微，很多时候都要加班。
在我离职前一天，我和我导师提出了我在看完编码规范的那几本书之后心中存在的一些问题，其中包括：
1、为什么Service层一个方法代码可以写五百行，而不是一个方法只处理一个功能
2、为什么NCC调用数据库的方式是最老式的JDBC，而不用Mybatis等工具
3、为什么不对一些有明显问题的代码进行重构
4、为什么代码中没有单元测试
导师的回答很详细，基本一个点对应一个点的给我讲解了。
首先让我看那几本书的作用是让我自己心里有一个标准，即知道代码应该怎样去写，类和方法应该怎样构建，要抽象共同方法来提高内聚、解除耦合。而其他程序员写代码时可能觉得这个方法能够返回想要的结果就行了，特别在老式的项目里，就基础会出现一个方法代码量极大的情况，另外想这些也是开发人员对于进程的妥协，因为项目周期赶得紧，很多时候没法仔细的去构建最完美的架构，只想着能够顺利完成就行了。单元测试和重构代码也同理，对于进度周期的妥协。
另外和我讲了NCC这个系统的可取之处，NCC诞生的时候，还没有Mybatis这门技术，另外NCC集成了七八种数据库，可以直接连接，如果要使用Mybatis的话需要些很多配置文件来实现数据源的适配。（大致是这个意思，其实我也不是很懂这个）

由衷的感谢我的第一位导师

此外还学了很多有用的东西，比如MyBatisPlus，这个东西在项目里面随处可见，很多新的SQL语句就是用这个写的，我临时学了一点，但是感觉没有太精通，就知道通过wrapper条件构造器可以快速进行SQL语句的调用。

最重要的收获可能就是编码规范和设计模式了，把两本书好好的过了一遍，对于编码规范和设计模式自己感觉熟悉了很多，可能没完全记下来，但是心里有一个大概了，现在编码的时候也会特别注意命名和设计等等。

注解：

@ResponseBody

@ResponseBody是作用在方法上的，@ResponseBody 表示该方法的返回结果直接写入 HTTP response body 中，一般在异步获取数据时使用【也就是AJAX】。
注意：在使用 @RequestMapping后，返回值通常解析为跳转路径，但是加上 @ResponseBody 后返回结果不会被解析为跳转路径，而是直接写入 HTTP response body 中。 比如异步获取 json 数据，加上 @ResponseBody 后，会直接返回 json 数据。@RequestBody 将 HTTP 请求正文插入方法中，使用适合的 HttpMessageConverter 将请求体写入某个对象。

@Deprecated

Java 中 @Deprecated 可以用来注解类、接口、成员方法和成员变量等，用于表示某个元素（类、方法等）已过时。当其他程序使用已过时的元素时，编译器将会给出警告。

@Documented

@Documented是元注解，可以修饰其他注解。许多注解头部都有@Documented注解，例如jdk中自带的@Deprecated注解(路径是：java/lang/Deprecated.java)头部就含有这个注解:
如果一个注解@B，被@Documented标注，那么被@B修饰的类，生成文档时，会显示@B。如果@B没有被@Documented标准，最终生成的文档中就不会显示@B。

@Retention

@Retention作用是定义被它所注解的注解保留多久，一共有三种策略，定义在RetentionPolicy枚举中.
从注释上看：
source：注解只保留在源文件，当Java文件编译成class文件的时候，注解被遗弃；被编译器忽略
class：注解被保留到class文件，但jvm加载class文件时候被遗弃，这是默认的生命周期
runtime：注解不仅被保存到class文件中，jvm加载class文件之后，仍然存在
这3个生命周期分别对应于：Java源文件(.java文件) —> .class文件 —> 内存中的字节码。

@Target

@Target说明了Annotation所修饰的对象范围：Annotation可被用于 packages、types（类、接口、枚举、Annotation类型）、类型成员（方法、构造方法、成员变量、枚举值）、方法参数和本地变量（如循环变量、catch参数）。在Annotation类型的声明中使用了target可更加明晰其修饰的目标。

@Inherited

@Inherited是一个标识，用来修饰注解
作用：如果一个类用上了@Inherited修饰的注解，那么其子类也会继承这个注解

注意：

接口用上个@Inherited修饰的注解，其实现类不会继承这个注解
父类的方法用了@Inherited修饰的注解，子类也不会继承这个注解

@SerializedName

SerializedName的主要作用：属性重命名，可以将json中的属性名转为我们自己自定义的属性名

@JsonProperty

此注解用于属性上，作用是把该属性的名称序列化为另外一个名称，如把trueName属性序列化为name，@JsonProperty(“name”)。
对属性名称重命名，比如在很多场景下Java对象的属性是按照规范的驼峰书写，但在数据库设计时使用的是下划线连接方式，此处在进行映射的时候就可以使用该注解。

@RequestMapping

RequestMapping是一个用来处理请求地址映射的注解，可用于类或方法上。用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。

RequestMapping注解有六个属性

value： 指定请求的实际地址，指定的地址可以是URI Template 模式（后面将会说明）；

method： 指定请求的method类型， GET、POST、PUT、DELETE等；

consumes： 指定处理请求的提交内容类型（Content-Type），例如application/json, text/html;

produces: 指定返回的内容类型，仅当request请求头中的(Accept)类型中包含该指定类型才返回；

params： 指定request中必须包含某些参数值是，才让该方法处理。

headers： 指定request中必须包含某些指定的header值，才能让该方法处理请求。

@RequestParam

@RequestParam用于将请求参数区数据映射到功能处理方法的参数上。

请求中包含username参数（如/requestparam1?userName=zhang），则自动传入。

@RequestParam注解主要参数：

value：参数名字，即入参的请求参数名字，如username表示请求的参数区中的名字为username的参数的值将传入；

required：是否必须，默认是true，表示请求中一定要有相应的参数，否则将报404错误码；

defaultValue：默认值，表示如果请求中没有同名参数时的默认值，默认值可以是SpEL表达式，如“#{systemProperties[‘java.vm.version’]}”。

@ExceptionHandler

用来统一处理方法抛出的异常

当使用这个@ExceptionHandler注解时，我们需要定义一个异常的处理方法，比如handleExeption()方法，给这个方法加上@ExceptionHandler注解，这个方法就会处理类中其他方法（被@RequestMapping注解）抛出的异常。

@RestControllerAdvice

@RestControllerAdvice是对Controller进行增强的，可以全局捕获spring mvc抛的异常。

即在Controller中的异常都会被捕获，无论是否用try-catch捕获

注：ControllerAdvice和RestControllerAdvice的区别
两者都是全局捕获异常，但是RestControllerAdvice更加强大，其作用相当于ControllerAdvice+ResponseBody

@Mapper

作用：在接口类上添加了@Mapper，在编译之后会生成相应的接口实现类
添加位置：接口类上面

@Mapper
public interface UserDAO {
//代码
}

如果想要每个接口都要变成实现类，那么需要在每个接口类上加上@Mapper注解，比较麻烦，解决这个问题用@MapperScan

@MapperScan

作用：指定要变成实现类的接口所在的包，然后包下面的所有接口在编译之后都会生成相应的实现类
添加位置：是在Springboot启动类上面添加。
