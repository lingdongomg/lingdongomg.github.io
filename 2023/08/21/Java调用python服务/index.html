ProcessBuilder:

ProcessBuilder 是 Java 5 引入的类，用于更灵活地创建和管理外部进程。它允许您设置进程的环境变量、工作目录、重定向输入输出流等。这使得您能够更好地控制外部进程的执行环境。下面是一个简单的示例：

Process process = Runtime.getRuntime().exec(“command arg1 arg2”);


优点：

简单易用，适用于基本的命令执行。

相对较少的代码量。

在性能方面，ProcessBuilder 通常被认为是更优的选择，因为它在创建和管理外部进程时提供了更多的控制选项，并且可以更有效地处理输入输出流。然而，性能的差异可能会在一些情况下非常微小，因此在许多情况下，您可能会选择基于您的代码结构和需求来进行选择。

总的来说，如果您需要更多的灵活性和控制，特别是需要重定向标准输入输出、设置环境变量或者切换工作目录，那么 ProcessBuilder 是更好的选择。如果您只是需要执行一个简单的外部命令，Runtime.getRuntime().exec() 也可以满足需求。

需要注意的是，自 Java 9 开始，ProcessBuilder 的改进使其在某些情况下更容易使用，因此在新的项目中，它更有可能是首选的方法。

## 后觉

方案确定下来之后，在后面的测试中发现，python中的print输出到控制台里面的内容，我通过ProcessBuilder能够在Java的输入输出流中获取到，也就是其实采用ProcessBuilder来调用Java，就不需要额外创建socket进行通信了。

另外如果采用socket进行通信，python里面就不要用print语句，因为如果Java中没有进行接受处理，会导致python程序输出缓冲区满，导致卡死无法执行下一步操作，这个bug当时还排查了好久。
