git&repo常用指令

git常用指令

初始化git

在从服务器拉去代码时，一般需要SSH公钥来授权，.pub 文件是公钥，另一个文件是密钥。用 ssh-keygen 的程序来建立它们。然后将公钥发送给服务器管理者即可。

git将本地项目上传到云端

关联远程仓库：

git remote add origin <远程仓库的URL>

推送到远程仓库：

git push -u origin master

取消之前添加的名为origin的远程仓库

git remote remove origin

这里需要git的用户名和邮箱和远程仓库匹配

如果与远程仓库有冲突，需要覆盖远程仓库的代码（一般用于新建的远程仓库），可以用以下命令强制推送：

git push -u origin master –force

git打补丁

git diff 为一个新增文件生成一个patch文件指令：

git diff –no-index /dev/null path/to/newfile > newfile.patch

解释：

  • git diff：用于比较当前工作目录与暂存区或已提交的版本之间的差异。
  • –no-index：用于告诉Git不要将当前目录看作是一个Git仓库。
  • /dev/null：空白的文件路径，表示一个空的初始版本。
  • path/to/newfile：新增文件的路径。
  • >：重定向符号，将输出结果保存到文件中。
  • newfile.patch：指定生成的patch文件的名称。

运行这条命令后，Git将比较一个空的初始版本（/dev/null）和新增文件的内容，并将差异写入名为newfile.patch的文件中。

git应用补丁：git apply 1.diff

git 回退操作

在发现当前版本有bug，需要回退到之前的版本时，执行的指令为：git revert commitId
git revert 的作用是创建一个新的提交，该提交取消之前某个提交引入的变更，从而将代码库恢复到指定提交的状态。这是通过应用指定提交的逆向补丁来实现的。
需要注意的是，git revert 不是修改历史的命令，而是通过创建新的提交来反转变更。因此，它是安全的，不会破坏历史记录。这也意味着你可以在分支上多次使用 git revert，每次撤销一个或多个提交。

除了git revert，还可以执行git reset --hard commitId

git reset 是 Git 中的一个命令，用于修改当前分支的指针位置，并可能修改工作目录和暂存区。git reset 的常见用法有三种模式：soft、mixed（默认）、和hard。

git reset --soft commitId 这种模式只是将 HEAD 指针移动到指定的提交，不会修改暂存区和工作目录。之后你可以通过修改文件、再次提交等方式，将代码库恢复到新的状态。

git reset commitId
这是默认的模式。它将 HEAD 指针移动到指定的提交，并且重置暂存区，但不会修改工作目录。这意味着工作目录的文件仍然保留对应提交的内容，但暂存区的文件将会回到指定提交的状态。

git reset --hard commitId
这种模式是最彻底的，它将 HEAD 指针移动到指定的提交，同时重置暂存区和工作目录，使它们完全匹配指定提交的状态。慎用，因为它会丢弃未提交的更改。

需要注意的是，git reset 是修改历史的命令，因此在对已共享的分支使用时要谨慎。如果你已经推送了这个分支，最好不要对已推送的提交执行 git reset 操作，以免破坏其他人的工作。在这种情况下，更安全的选择可能是使用 git revert 创建一个新的撤销提交。

如果你通过 git reset 将指针移动到指定的提交，并发现后续的提交记录不可见，可以执行git reflog 命令，它将显示 HEAD 的历史记录，包括之前的位置和提交。

git删除操作

在仓库进行修改后想要不保存修改还原，可以采用 git checkout .
在仓库中新增了文件并没有进行跟踪(即没有进行git add)，此时想要将未跟踪文件删除可以使用 git clean -df
注意该操作不可撤销，删除后就无法找回文件了，要查看git clean会删除哪些文件可以先使用 git clean -n查看将要被删除的文件，并不实际删除文件

repo

repo是一个用python脚本来统一管理若干个项目的git仓库，使git的使用更简单。安卓开发一般采用repo进行版本管理。

.repo为版本管理文件夹，其中有安卓的各个版本的镜像。

Repo的安装

curl http://commondatastorage.googleapis.com/git-repo-downloads/repo > ~/bin/repo

chmod a+x ~/bin/repo

创建ssh密钥

sudo apt-get install openssh-client

ssh-keygen -t rsa -C “your_name@techvision.com.cn“

把生成的xxx_rsa.pub发给代码管理者
